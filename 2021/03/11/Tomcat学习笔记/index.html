0x03 JSP 与 Servlet

3.0 JSP简介 & 生命周期

JSP是一种用于编写动态网页的语言，可以类比于PHP，通过特殊的语法使得其能够在HTML里编写Java代码。不过由于目前MVC架构越来越流行，所以JSP也用的越来越少了，不过在一些老的项目中还是会用到JSP，因为其重构成本太高，所以JSP还是有学习的必要的。

在了解JSP这门语言之前，不妨先了解一下能够解析JSP的服务器（Tomcat、Resin等）处理JSP的流程：

  • 用户通过浏览器访问服务器资源
  • 服务器识别到用户访问的资源为JSP文件，开始对其进行解析
  • JSP引擎从磁盘中加载JSP文件，并通过Jasper将其转换为Java源码，这些源码文件被称为Servlet，后文将学习到
  • JSP引擎将Servlet编译为类（.class文件），并将原始请求传递给Servlet引擎
  • Web服务器调用Servlet引擎，载入并执行此Servlet类，在执行过程中产生的输出会输出到Response中

JSP的生命周期与Servlet类似，都是由创建到销毁的整个过程，从上述步骤中可以看出其区别是多了一个将JSP转换为Servlet的步骤：

  • 编译阶段
    • servlet容器编译servlet源文件，生成servlet类
  • 初始化阶段
    • 加载与JSP对应的servlet类，创建其实例，并调用它的初始化方法
  • 执行阶段
    • 调用与JSP对应的servlet实例的服务方法
  • 销毁阶段
    • 调用与JSP对应的servlet实例的销毁方法，然后销毁servlet实例

编译阶段：当浏览器请求服务器资源（JSP文件）时，JSP引擎会判断此资源是否未被编译，或是否相对于上一次编译进行了更改，如符合上述两种情况，则会对其重新编译。

3.2 Servlet Filter

在开发WEB服务的过程中，必不可少的就是Filter，在日后审计代码时也会经常遇到，其主要起以下作用：

  • 在客户端的请求访问后端资源之前，拦截这些请求。
  • 在服务器的响应发送回客户端之前，处理这些响应。

Filter是一个实现了javax.servlet.Filter接口的类，此接口主要包含以下几个方法：

序 方法 & 描述
1 public void doFilter (ServletRequest, ServletResponse, FilterChain) 该方法完成实际的过滤操作，当客户端的请求与过滤器设置的 URL 匹配时，Servlet 容器将先调用过滤器的 doFilter 方法。FilterChain 用于访问后续过滤器。
2 public void init(FilterConfig filterConfig) web 应用程序启动时，web 服务器将创建Filter 的实例对象，并调用其init方法，读取web.xml配置，完成对象的初始化功能，从而为后续的用户请求作好拦截的准备工作（filter对象只会创建一次，init方法也只会执行一次）。开发人员通过init方法的参数，可获得代表当前filter配置信息的FilterConfig对象。
3 public void destroy() Servlet容器在销毁过滤器实例前调用该方法，在该方法中释放Servlet过滤器占用的资源。

除了上面所记录的，web.xml还支持控制Filter的作用对象，通过dispatcher标签实现：

  • REQUEST：当用户直接访问页面时，Web容器将会调用过滤器。如果目标资源是通过RequestDispatcher的include()或forward()方法访问时，那么该过滤器就不会被调用。
  • INCLUDE：如果目标资源是通过RequestDispatcher的include()方法访问时，那么该过滤器将被调用。除此之外，该过滤器不会被调用。
  • FORWARD：如果目标资源是通过RequestDispatcher的forward()方法访问时，那么该过滤器将被调用，除此之外，该过滤器不会被调用。
  • ERROR：如果目标资源是通过声明式异常处理机制调用时，那么该过滤器将被调用。除此之外，过滤器不会被调用。

0x04 web.xml中的常见配置项

4.0 context-param

该元素用来声明应用范围(整个WEB项目)内的上下文初始化参数。

4.1 display-name

该元素用于标记整个WEB项目的名称。

4.2 filter

该元素用于标记Filter，定义其名称以及对应的class和初始化此Filter所使用的参数。

4.3 filter-mapping

该元素用于声明一个Filter所匹配的url，通常使用正则表达式进行匹配。

4.4 listener

该元素用于声明一个Listener，其实现了javax.servlet.ServletContextListener 接口的服务器端程序，随web应用的启动而启动，只初始化一次，随web应用的停止而销毁。主要作用是： 做一些初始化的内容添加工作、设置一些基本的内容、比如一些参数或者是一些固定的对象等等。

4.5 servlet

该元素用于声明一个Servlet，用于后续的Mapping。

4.6 servlet-mapping

该元素用于指定一个servlet在何时被访问，与Filter类似。

4.7 error-page

该元素用于指定当遇到某错误时对应的处理。

4.8 resource-ref

该元素用于配制某些在上下文中可以直接取的连接对象。
